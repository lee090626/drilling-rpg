# 커밋 컨벤션 (Commit Convention)

이 프로젝트는 코드의 가독성과 히스토리 관리를 위해 **Conventional Commits** 표준을 따릅니다.

## 메시지 형식

```text
<type>(<scope>): <subject>

<body>
```

- **type**: 커밋의 성격을 나타냅니다. (필수)
- **scope**: 변경된 부분이나 모듈을 나타냅니다. (선택)
- **subject**: 변경 사항에 대한 짧은 요약입니다. (필수)
- **body**: 변경 동기, 상세 내용 및 이전과의 차이점을 기록합니다. (상세 내용 필요 시)

## 커밋 타입 (Type)

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정 (렌더링 오류, 로직 오류 등)
- `docs`: 문서 수정 (README.md, 주석 추가 등)
- `style`: 코드 형식, 세미콜론 누락 등 (로직 변경 없음)
- `refactor`: 코드 리팩토링 (기능 변화 없이 코드 구조 개선)
- `test`: 테스트 코드 추가 및 수정
- `chore`: 빌드 업무 수정, 패키지 매니저 설정 등 (프로덕션 코드 변경 없음)
- `perf`: 성능 개선

## 예시

```text
feat(mining): 드릴 업그레이드 기능 추가
fix(render): 베이스캠프 타일 오프셋 렌더링 오류 수정
docs(script): 아틀라스 생성 스크립트 한국어 주석 추가
```
